# Отчет об исправлении проблем с кодировкой в скриптах Windows

## Обзор проблемы

При выполнении Windows batch скриптов в проекте HyperEngine обнаружены проблемы с отображением русского текста. Символы отображались как нечитаемые последовательности (например, "╨Ч╨░╨┐╤Г╤Б╨║" вместо "Запуск").

### Причина проблемы

Проблема была вызвана несоответствием кодировок:
- **Файлы скриптов**: сохранены в кодировке UTF-8
- **Windows Command Prompt**: по умолчанию использует кодировку Windows-1251 (CP1251)

## Исправленные файлы

Добавлена команда `chcp 65001 >nul` в начало следующих batch файлов для переключения консоли в режим UTF-8:

1. **scripts/static_analysis.bat** - скрипт статического анализа кода
2. **scripts/quality_check.bat** - скрипт проверки качества кода  
3. **scripts/format_code.bat** - скрипт форматирования кода
4. **scripts/setup_git_hooks.bat** - скрипт настройки git hooks
5. **scripts/check_format_simple.bat** - простая проверка форматирования
6. **scripts/fix_formatting.bat** - автоматическое исправление форматирования

## Примененное решение

### До исправления:
```batch
@echo off
REM Комплексный статический анализ кода HyperEngine для Windows
setlocal enabledelayedexpansion
```

### После исправления:
```batch
@echo off
chcp 65001 >nul
REM Комплексный статический анализ кода HyperEngine для Windows
setlocal enabledelayedexpansion
```

### Объяснение команды `chcp 65001 >nul`:
- `chcp 65001` - переключает кодовую страницу консоли на UTF-8
- `>nul` - подавляет вывод сообщения о смене кодировки

## Рекомендации

### Для будущих скриптов:
1. **Всегда добавляйте** `chcp 65001 >nul` в начало batch файлов с русским текстом
2. **Сохраняйте файлы** в кодировке UTF-8 для поддержки Unicode символов
3. **Тестируйте скрипты** на чистой Windows системе перед коммитом

### Альтернативные решения:
1. **PowerShell скрипты** - нативно поддерживают UTF-8
2. **Bash скрипты** - для использования в Git Bash или WSL
3. **Только английский текст** - для максимальной совместимости
