## 🧪 Тестирование PR

После создания PR рекомендуется:

1. **Проверить CI/CD:**
   - Убедиться что новые job'ы запускаются
   - Проверить отсутствие ошибок сборки
   - Просмотреть артефакты с отчетами

2. **Локальное тестирование:**
   ```bash
   # Проверить форматирование
   find src include tests -name "*.cpp" -o -name "*.h" | \
   xargs clang-format --dry-run --Werror --style=file
   
   # Запустить комплексную проверку (если инструменты установлены)
   ./scripts/quality_check.sh
   ```

3. **Проверить pre-commit hooks:**
   ```bash
   # Установить pre-commit (если не установлен)
   pip install pre-commit
   
   # Установить hooks
   pre-commit install
   
   # Запустить на всех файлах
   pre-commit run --all-files
   ```
