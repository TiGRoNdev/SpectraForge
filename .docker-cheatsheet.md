# 🐳 Docker Quick Commands Cheatsheet

## 🚀 Quick Start (Copy-Paste Ready)

### First Time Setup
```bash
# Build CI/CD image
docker-compose build ci-runner

# Verify installation
docker run --rm hyperengine-ci:latest cmake --version
```

### Daily Development
```bash
# Start development environment
docker-compose up -d dev && docker exec -it hyperengine-dev bash

# Run quality checks
docker-compose run --rm ci-runner ./scripts/quality_check.sh

# Build project
docker-compose run --rm ci-runner bash -c "cmake -B build -G Ninja && cmake --build build -j\$(nproc)"

# Run tests
docker-compose run --rm ci-runner bash -c "cd build && ctest --output-on-failure"
```

## 🔧 Code Quality

### Format Code
```bash
# Check formatting
docker-compose run --rm ci-runner \
  find src include -name "*.cpp" -o -name "*.h" | \
  xargs clang-format --dry-run --Werror

# Fix formatting
docker-compose run --rm ci-runner \
  find src include -name "*.cpp" -o -name "*.h" | \
  xargs clang-format -i
```

### Static Analysis
```bash
# clang-tidy
docker-compose run --rm ci-runner bash -c "
  cmake -B build -DCMAKE_EXPORT_COMPILE_COMMANDS=ON
  find src -name '*.cpp' | xargs clang-tidy -p build
"

# cppcheck
docker-compose run --rm ci-runner \
  cppcheck --enable=all --std=c++20 src/ include/
```

### Memory Check
```bash
# Valgrind
docker-compose run --rm ci-runner bash -c "
  cmake -B build -DCMAKE_BUILD_TYPE=Debug
  cmake --build build
  valgrind --leak-check=full ./build/tests/unit_tests
"
```

## 🧪 Testing

### All Tests
```bash
docker-compose run --rm ci-runner bash -c "cd build && ctest"
```

### Specific Test
```bash
docker-compose run --rm ci-runner bash -c "cd build && ctest -R TestName"
```

### Verbose Output
```bash
docker-compose run --rm ci-runner bash -c "cd build && ctest --verbose"
```

## 🎮 Demo

### Run GUI Demo (Linux with X11)
```bash
xhost +local:docker
docker-compose up demo
xhost -local:docker
```

## 🧹 Cleanup

### Stop All Containers
```bash
docker-compose down
```

### Remove Volumes (Clean Build)
```bash
docker-compose down -v
```

### Full Cleanup
```bash
docker-compose down --rmi all -v
docker system prune -a
```

## 📊 Diagnostics

### Check Container Status
```bash
docker-compose ps
```

### View Logs
```bash
docker-compose logs ci-runner
```

### Interactive Shell
```bash
docker exec -it hyperengine-dev bash
```

### Check ccache Stats
```bash
docker-compose run --rm ci-runner ccache -s
```

## 🔄 Update

### Rebuild Without Cache
```bash
docker-compose build --no-cache ci-runner
```

### Pull Latest Base Images
```bash
docker-compose build --pull ci-runner
```

## 💡 Tips

- Use `--rm` to auto-remove containers after exit
- Use `-d` to run in background
- Use `-v` for verbose output
- Use `$(nproc)` for parallel builds
- Named volumes persist between runs
