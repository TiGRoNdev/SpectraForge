# Руководство по вкладу в проект HyperEngine

Спасибо за интерес к проекту HyperEngine! Мы приветствуем вклад от сообщества и ценим любую помощь в развитии современного 3D/4D игрового движка.

## 🚀 Как внести вклад

### Сообщение об ошибках

Если вы нашли ошибку, пожалуйста:

1. Проверьте, не была ли уже сообщена эта ошибка в [Issues](https://github.com/yourusername/hyperengine/issues)
2. Создайте новый issue с подробным описанием:
   - Описание проблемы
   - Шаги для воспроизведения
   - Ожидаемое поведение
   - Фактическое поведение
   - Скриншоты (если применимо)
   - Информация о системе (ОС, версия компилятора, etc.)

### Предложение новых функций

Для предложения новых функций:

1. Создайте issue с тегом "enhancement"
2. Опишите предлагаемую функцию
3. Объясните, как она вписывается в архитектуру движка
4. Предоставьте примеры использования

### Внесение изменений в код

1. **Fork** репозитория
2. Создайте новую ветку для ваших изменений:
   ```bash
   git checkout -b feature/your-feature-name
   # или
   git checkout -b fix/your-bug-fix
   ```
3. Внесите изменения
4. Добавьте тесты (если применимо)
5. Обновите документацию
6. Убедитесь, что код компилируется и тесты проходят
7. Создайте Pull Request

## 📋 Стандарты кодирования

### C++ Стиль

Мы следуем стандартам C++17 и используем следующие соглашения:

```cpp
// Имена классов в PascalCase
class GameObject4D {
public:
    // Публичные методы в camelCase
    void setPosition(const Vector4& position);
    
private:
    // Приватные члены с префиксом m_
    Vector4 m_position;
    bool m_isActive;
};

// Имена функций в camelCase
void updateGameObjects(float deltaTime);

// Константы в UPPER_CASE
const float MAX_VELOCITY = 100.0f;

// Enum классы в PascalCase
enum class ProjectionMode {
    Orthographic,
    Perspective,
    CrossSection
};
```

### Комментарии

- Используйте комментарии на русском языке
- Документируйте публичные API
- Объясняйте сложную логику
- Используйте `//` для однострочных комментариев
- Используйте `/* */` для многострочных комментариев

```cpp
/**
 * @brief 4-мерный вектор для работы с гиперпространством
 * 
 * Представляет точку или направление в 4D пространстве с координатами [x, y, z, w]
 * где w - четвертое измерение (гиперпространство)
 */
class Vector4 {
public:
    // Конструкторы
    Vector4();                                    // [0, 0, 0, 0]
    Vector4(float x, float y, float z, float w); // [x, y, z, w]
    
    // Математические операции
    float magnitude() const;                      // Длина вектора
    Vector4 normalized() const;                   // Нормализованный вектор
};
```

## 🧪 Тестирование

### Типы тестов

1. **Unit тесты**: Тестирование отдельных функций и классов
2. **Integration тесты**: Тестирование взаимодействия компонентов
3. **Performance тесты**: Тестирование производительности

### Пример unit теста

```cpp
#include <gtest/gtest.h>
#include <Engine4D/Math/Vector4.h>

TEST(Vector4Test, Addition) {
    Vector4 a(1, 2, 3, 4);
    Vector4 b(5, 6, 7, 8);
    Vector4 result = a + b;
    
    EXPECT_FLOAT_EQ(result.x, 6.0f);
    EXPECT_FLOAT_EQ(result.y, 8.0f);
    EXPECT_FLOAT_EQ(result.z, 10.0f);
    EXPECT_FLOAT_EQ(result.w, 12.0f);
}

TEST(Vector4Test, Magnitude) {
    Vector4 v(3, 4, 0, 0);
    EXPECT_FLOAT_EQ(v.magnitude(), 5.0f);
}
```

### Запуск тестов

```bash
cd build
ctest --verbose
```
