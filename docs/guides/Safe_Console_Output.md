# Безопасный вывод в консоль

## Проблема

На Windows системах демо-приложения могут прерываться при выводе символов, которые консоль не может корректно отобразить. Это происходит из-за проблем с кодировкой UTF-8, особенно при использовании эмодзи и специальных символов.

## Решение

Мы добавили систему безопасного вывода в класс `Console`, которая автоматически обрабатывает проблемы с кодировкой и предотвращает сбои приложений.

## Основные возможности

### 1. Безопасное преобразование чисел

```cpp
// Вместо std::to_string(value)
std::cout << "Значение: " << Console::safeToString(42) << std::endl;

// Или через макрос
std::cout << "Значение: " << SAFE_TO_STRING(42) << std::endl;
```

### 2. Безопасный вывод текста

```cpp
// Базовый безопасный вывод
Console::safePrint("Текст с эмодзи 🚀");

// Безопасный вывод с переносом строки
Console::safePrintLine("Сообщение");

// Безопасный вывод с fallback
Console::safePrint("Проблемный текст", "Резервный текст");
```

### 3. Безопасное логирование

```cpp
// Методы класса
Console::safeInfo("Информационное сообщение");
Console::safeWarning("Предупреждение");
Console::safeError("Ошибка");

// Или через макросы
SAFE_INFO("Информация");
SAFE_WARNING("Предупреждение");
SAFE_ERROR("Ошибка");
```

### 4. Проверка и очистка текста

```cpp
std::string text = "Текст с символами ♠ ♣ ♥ ♦";

// Проверяем безопасность
if (Console::isTextSafe(text)) {
    std::cout << text << std::endl;
} else {
    // Очищаем от проблемных символов
    std::string cleanText = Console::sanitizeText(text);
    std::cout << cleanText << std::endl;
}
```

## Доступные макросы

```cpp
// Безопасный вывод
SAFE_PRINT(text)                    // Вывод без переноса
SAFE_PRINT_LINE(text)              // Вывод с переносом
SAFE_PRINT_FALLBACK(text, fallback) // Вывод с резервным текстом

// Преобразование чисел
SAFE_TO_STRING(value)               // Безопасное std::to_string

// Логирование
SAFE_INFO(msg)                      // Информация
SAFE_WARNING(msg)                   // Предупреждение
SAFE_ERROR(msg)                     // Ошибка
```

## Миграция существующего кода

### Было:
```cpp
std::cout << "Значение: " << std::to_string(value) << std::endl;
std::cout << "🚀 Запуск приложения..." << std::endl;
```

### Стало:
```cpp
std::cout << "Значение: " << Console::safeToString(value) << std::endl;
Console::safePrintLine("🚀 Запуск приложения...");
```

## Как это работает

1. **Проверка символов**: Система проверяет каждый символ в строке на корректность UTF-8
2. **Обнаружение эмодзи**: Специальная обработка 4-байтовых UTF-8 последовательностей
3. **Автоматический fallback**: При ошибке используется очищенная версия или резервный текст
4. **Graceful degradation**: В крайнем случае выводится базовое ASCII сообщение

## Пример использования

```cpp
#include "Engine3D/Core/Console.h"

int main() {
    // Инициализация консоли
    Console::initialize();
    
    // Безопасный вывод
    Console::safeInfo("Приложение запущено");
    
    int frameCount = 1000;
    Console::safePrintLine("Кадров отрендерено: " + Console::safeToString(frameCount));
    
    // Проблемный текст будет обработан автоматически
    Console::safePrintLine("Статус: ✅ Готово");
    
    return 0;
}
```

## Рекомендации

1. **Всегда используйте** `Console::safeToString()` вместо `std::to_string()`
2. **Предпочитайте** `Console::safePrintLine()` для вывода сообщений
3. **Используйте макросы** для краткости кода
4. **Инициализируйте консоль** в начале программы через `Console::initialize()`
