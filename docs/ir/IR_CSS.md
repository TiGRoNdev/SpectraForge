### 1. **DEADLOCK ПРИ SHUTDOWN - АРХИТЕКТУРНАЯ ПРОБЛЕМА СИНХРОНИЗАЦИИ**

**Обнаружено**: Полное отсутствие синхронизации между CPU и GPU при уничтожении ресурсов.

**Анализ архитектуры**:

#### ❌ `src/core/VulkanContextImpl.cpp:207-229` - Критическая ошибка
```cpp
void cleanup() {
    if (!initialized_) return;

    // ❌ НЕТ СИНХРОНИЗАЦИИ - GPU может выполнять команды!
    core::VMAMemoryManager::getInstance().cleanup();

    if (device_) {
        device_.destroy();  // Уничтожает device пока GPU занят
    }

    if (instance_) {
        instance_.destroy();
    }
}
```

**Последствия**:
- GPU выполняет команды рендеринга в фоновом режиме
- CPU начинает уничтожать ресурсы (buffers, images, pipelines)
- GPU пытается обратиться к уже освобожденной памяти
- **Deadlock**: CPU ждет GPU, GPU ждет ресурсы → система зависает
- OOM Killer убивает процесс через ~20 секунд

#### ❌ `src/app/Engine.cpp:539-554` - Каскадная проблема
```cpp
void Engine::shutdown() {
    // ❌ НЕТ проверки завершения GPU операций
    if (scene_manager_) {
        scene_manager_->shutdown();  // Может запустить GPU команды
        scene_manager_.reset();
    }
    if (core_) {
        core_->shutdown();  // Внутри вызывается renderer_->shutdown()
        core_.reset();
    }
    // Уничтожение ресурсов без ожидания
}
```

**Архитектурное решение**:
```cpp
void Engine::shutdown() {
    // ✅ ОБЯЗАТЕЛЬНАЯ СИНХРОНИЗАЦИЯ
    if (renderer_ && renderer_->isInitialized()) {
        std::cout << "[Engine] 🔄 Синхронизация с GPU...\n";
        // Ждем завершения всех операций GPU
        vk::Device device = getVulkanDevice();
        device.waitIdle();  // <-- КРИТИЧНО!
    }

    if (scene_manager_) {
        scene_manager_->shutdown();
        scene_manager_.reset();
    }
    if (core_) {
        core_->shutdown();
        core_.reset();
    }
    if (window_) {
        window_->close();
        window_.reset();
    }
    SpectraForge::Core::Window::terminateSystem();

    std::cout << "[Engine] ✅ Shutdown complete\n";
}
```

### 3. **ЧЕРНЫЙ ЭКРАН - АЛГОРИТМИЧЕСКИЙ ДЕФЕКТ SDF**

**Анализ алгоритма**: `shaders/TriangleSplatting.comp:57-135`

#### ❌ Критическая ошибка в SDF алгоритме

**Текущий алгоритм**:
```glsl
float computeTriangleSDF(vec2 p, vec2 v0, vec2 v1, vec2 v2) {
    float d0 = halfPlaneDistance(p, v0, v1);  // Расстояние до ребра 0
    float d1 = halfPlaneDistance(p, v1, v2);  // Расстояние до ребра 1
    float d2 = halfPlaneDistance(p, v2, v0);  // Расстояние до ребра 2

    // ❌ ОШИБКА: max(max(d0, d1), d2) всегда положительный или нулевой
    return max(max(d0, d1), d2);  // Всегда >= 0!
}
```

**Проблема**: Алгоритм возвращает только максимальное расстояние до ребер, что всегда положительно или нулевое. Нет отрицательных значений внутри треугольника!

**Корректный алгоритм**:
```glsl
float computeTriangleSDF(vec2 p, vec2 v0, vec2 v1, vec2 v2) {
    // Все расстояния должны быть отрицательными внутри треугольника
    float d0 = halfPlaneDistance(p, v0, v1);  // Отрицательное внутри
    float d1 = halfPlaneDistance(p, v1, v2);  // Отрицательное внутри
    float d2 = halfPlaneDistance(p, v2, v0);  // Отрицательное внутри

    // ✅ ПРАВИЛЬНО: max всех расстояний
    // Если все отрицательные → внутри треугольника
    // Если хотя бы одно положительное → снаружи
    return max(max(d0, d1), d2);
}
```

**Тестовый случай**:
```glsl
// Для точки внутри треугольника:
// d0 = -1.5, d1 = -2.1, d2 = -0.8
// max(-1.5, -2.1, -0.8) = -0.8 → Внутри (отрицательное)

// Для точки снаружи:
// d0 = 1.2, d1 = -0.5, d2 = 2.1
// max(1.2, -0.5, 2.1) = 2.1 → Снаружи (положительное)
```

**Последствия**:
- Все пиксели имеют `phi > 0` (снаружи)
- Условие `if (phi > 0.0) continue;` пропускает ВСЕ пиксели
- Результат: полностью черный экран

### 4. **СОРТИРОВКА ПО ГЛУБИНЕ - АЛГОРИТМИЧЕСКИЙ ДЕФЕКТ**

**Анализ алгоритма**: `src/rendering/TriangleSplattingPass.cpp:894-950`

#### ❌ Критическая ошибка в алгоритме сортировки

**Текущий алгоритм**:
```cpp
void sortTrianglesByDepth(vk::CommandBuffer cmd) {
    // Compute depth keys for all triangles
    computeDepthKeys(cmd, cameraPosition_);

    // Execute bitonic sort
    // ❌ ПРОБЛЕМА: Сортировка по depth keys НЕ означает сортировку по глубине!
    // Depth key может быть неправильным или не соответствовать визуальной глубине
}
```

**Проблема**: Алгоритм сортирует по `depth keys`, но не учитывает перспективную проекцию и визуальную глубину треугольников.

**Корректный алгоритм**:
```cpp
void computeDepthKeys(vk::CommandBuffer cmd, const glm::vec3& cameraPosition) {
    // ✅ ПРАВИЛЬНО: Вычислять визуальную глубину каждого треугольника
    // Учитывать перспективу и расстояние до камеры
    for (each triangle) {
        // Центроид треугольника в world space
        vec3 centroid = (v0 + v1 + v2) / 3.0f;

        // Визуальная глубина = расстояние от камеры к плоскости треугольника
        // Или проекция центра на ray камеры
        float depth = dot(centroid - cameraPosition, cameraDirection);
        depthKeys[i] = depth;
    }
}
```

**Текущие проблемы сортировки**:
1. **Неправильные depth keys** - могут не соответствовать визуальной глубине
2. **Отсутствие стабильной сортировки** - одинаковые глубины могут менять порядок
3. **Нет учета окклюзии** - треугольники могут перекрываться неправильно

## 📊 АНАЛИЗ ПРОИЗВОДИТЕЛЬНОСТИ - ГЛУБОКИЙ ВЗГЛЯД

### Метрики сложности

| Алгоритм | Сложность | Фактическая | Целевая | Ускорение нужно |
|----------|-----------|-------------|---------|----------------|
| **Triangle Splatting** | O(N×M) | 83 млрд операций | 100 млн операций | **830×** |
| **Gaussian Splatting** | O(N×log N) | ~1 млн операций | 100 млн операций | Нет проблем |
| **Rasterization** | O(N) | ~1 млн операций | 100 млн операций | Нет проблем |

### Bottleneck анализ

**GPU Memory Bandwidth**:
- Чтение `triangleBuffer_`: 40,211 × 64 bytes = ~2.5 MB на пиксель
- Чтение `sortedIndicesBuffer_`: 40,211 × 4 bytes = ~160 KB на пиксель
- **Итого**: ~2.7 MB × 2 млн пикселей = **5.4 TB** на кадр!

**Compute Occupancy**:
- Каждый SM занят обработкой одного пикселя
- Нет параллелизма между пикселями
- 100% GPU утилизация, но низкая эффективность
