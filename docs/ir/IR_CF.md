# IR_CF — Углублённый алгоритмический отчёт (Core Findings)

Дата: 2025‑10‑04
Объект: `examples/SpectraForge_Example_Demo.cpp` + `HybridFreGSRenderer` + `TriangleSplattingPass` + `VulkanContext`
Цель: Устранить чёрный экран и зависание при ESC; стабильно работать без риска для системы

---

## 1) Root Causes (корневые причины)

- R1. Vulkan shutdown без device.waitIdle(): отсутствует ожидание завершения GPU перед разрушением ресурсов (`src/core/VulkanContextImpl.cpp:207-229`).
- R2. O(N×M) пиксель‑к‑треугольнику в `TriangleSplatting.comp` (каждый пиксель итерирует все треугольники) → 83 млрд итераций/кадр @1080p.
- R3. SDF/Window: при ошибочных знаках в half‑plane тесте все пиксели проходят как outside → чёрный экран.
- R4. Depth sort: ключ глубины не перспективно‑корректен; порядок может нарушаться (артефакты/время). 
- R5. Presentation path: требуется строгий порядок барьеров `TRANSFER_DST → PRESENT_SRC` и корректная обработка `OUT_OF_DATE/SUBOPTIMAL`.

---

## 2) Верификация (что подтверждено в репо)

- V1. Нет `device.waitIdle()` в `VulkanContextImpl::cleanup()` (grep по workspace — отсутствует).
- V2. В `TriangleSplatting.comp` (165–234) — внешний цикл по всем треугольникам для каждого пикселя.
- V3. PNG кадр сохраняется, но размер 80 KB для 1920×1080 указывает на низкую энтропию (высокий риск почти чёрного контента).
- V4. Барьеры и present в `HybridFreGSRenderer::renderFrame` выглядят корректно, но отсутствие быстрого early‑out делает pipeline перегруженным.

---

## 3) Диагностические проверки (минимальные шаги)

- D1. Добавить временный debug в шейдер: принудительная заливка `imageStore(..., vec4(1,0,0,1))` перед итоговой записью для 1 кадра.
- D2. Лог VP‑матрицы в `HybridFreGSRenderer` (первые 2 кадра) и проверка frustum.
- D3. Временно понизить разрешение до 960×540 + сократить треугольники (step≈tris/5000) — проверить, уходит ли 5‑секундный кадр.
- D4. Включить валидационные слои Vulkan и проверить сообщения о sync/layout.

## 4) План фиксов (по приоритету)

- F1 (критично). Добавить `device_.waitIdle()` в `VulkanContextImpl::cleanup()` перед VMA cleanup и destroy device.
- F2 (критично). Шейдер: ввести AABB early‑out для треугольника по pixelPos; снизит отбор до релевантных треугольников.
- F3 (критично). Tile‑based culling: предварительный compute pass собирает списки видимых треугольников на тайл (16×16); основной pass читает только списки тайла.
- F4 (высокий). Пересчитать depth key перспективно‑корректно (центроид → view‑space z; стабильная сортировка).
- F5 (средний). В демо: 960×540 и `step = max(1, tris/5000)` как быстрый траблшутинг; вернуть позже.

## 5) Соответствие Triangle Splatting и Renderer.md (кратко)

- T1. Окно на SDF: корректный знак half‑plane и нормалей по часовой/против часовой — критично; при ошибке → чёрный экран.
- T2. Требуется spatial отбор (binning/culling); полный перебор противоречит целевым требованиям `Renderer.md` (низкая асимптотика).
- T3. Барьеры и present соответствуют схеме; узкое место — подготовка данных (visibility/depth) до present.

## 6) Метрики успеха (после F1–F3)

- M1. Завершение: < 1 сек, без Killed.
- M2. Кадр: ≤ 100–150 ms @960×540 (до полноценного culling) → FPS ≥ 10.
- M3. Чёрный экран исчезает при debug‑заливке; затем появляется геометрия при включении SDF.
