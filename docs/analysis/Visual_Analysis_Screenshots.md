## 📸 Скриншот 1: "Улыбка" (контуры арок)

### Описание
```
┌───────────────────────────────────────┐
│                                       │
│         ●  ●  ●                       │
│      ●              ●                 │
│    ●                  ●               │
│   ●                    ●              │
│    ●                  ●               │
│      ●              ●                 │
│         ●  ●  ●                       │
│                                       │
│                  ●  ●  ●  ●           │
│                ●            ●         │
│               ●              ●        │
└───────────────────────────────────────┘
```

### Анализ паттерна

**Наблюдение 1: Две арки**
- Верхняя арка: контур "улыбки" из ~30 точек
- Нижняя арка: фрагментарный контур из ~20 точек

**Наблюдение 2: Расположение точек**
- Точки **на контуре**, НЕ внутри
- Расстояние между точками: ~10-15 пикселей
- Размер точек: ~2-3 пикселя (видимый радиус)

**Наблюдение 3: Геометрия**
- Форма соответствует **аркам Sponza** (атриум)
- Видна **перспектива** (ближние точки крупнее)
- **Z-ordering** работает (дальние арки позади)

### Математическая интерпретация

**Почему видны только контуры?**

1. **Барицентрическое семплирование**:
   ```
   Треугольник арки:
       v0 ───────── v1
        │         ╱
        │       ╱
        │     ╱
        │   ╱
        │ ╱
       v2
   
   Наши точки:
   - Барицентр: (v0 + v1 + v2) / 3  ← ВНУТРИ
   - Mid edges:  (v0+v1)/2, (v1+v2)/2, (v2+v0)/2  ← НА РЁБРАХ
   ```
   
   Результат: Точки **концентрируются у рёбер** треугольника, формируя контур.

2. **Subdivision 9 точек**:
   ```
   Для большого треугольника (area > 0.08):
   
   v0 ────── q0 ─── m01 ─── q1 ───── v1
    │                                │
   m20      in01          in12      m12
    │                                │
    │          bc (центр)            │
    │                                │
   q2                                q2'
    │                                │
   v2 ──────────────────────────────  
   
   9 точек, но НЕ регулярная сетка!
   ```

3. **Визуализация плотности**:
   ```
   Треугольник 300×200 px (60,000 пикселей):
   
   Наше покрытие:
   ●                     ← 9 точек × 8 px² = 72 px покрыто
                         ← 59,928 px НЕ покрыто
   Покрытие: 0.12%
   ```

**Вывод**: Видим "скелет" сцены, но не "кожу".

## 📸 Скриншот 2: Усиленная плотность

### Описание
```
┌───────────────────────────────────────┐
│                                       │
│     ● ● ● ● ● ●                       │
│   ●  ●  ●  ●  ●  ●                    │
│  ●    ●        ●    ●                 │
│ ●      ●      ●      ●                │
│  ●    ●        ●    ●                 │
│   ●  ●  ●  ●  ●  ●                    │
│     ● ● ● ● ● ●                       │
│                                       │
│              ● ● ● ● ● ● ●            │
│            ●  ●  ●  ●  ●  ●           │
│           ●    ●        ●    ●        │
└───────────────────────────────────────┘
```

### Изменения относительно скриншота 1

**Квантитативные изменения**:
- Точек стало больше: ~30 → ~80 на арку (+167%)
- Расстояние между точками: 10-15 px → 5-8 px (-46%)
- Размер точек увеличен: 2-3 px → 3-4 px (+33%)

**Качественные изменения**:
- ✅ Контуры **более детальные**
- ✅ Видны **дополнительные элементы** (колонны по бокам)
- ❌ Всё ещё **НЕ заполнены** поверхности
- ❌ Видны **просветы** между точками

### Математическая интерпретация

**Увеличение плотности**:
```
Было:
  - 66,000 треугольников
  - step = tris.size() / 15,000 → обрабатываем ~15,000
  - subdivision avg ≈ 2.4
  - Итого: 36,000 гауссианов

Стало:
  - 66,000 треугольников
  - step = tris.size() / 50,000 → обрабатываем ~50,000
  - subdivision avg ≈ 4.5
  - Итого: 225,000 гауссианов (+520%)
```

**Покрытие экрана**:
```
Экран: 1920 × 1080 = 2,073,600 пикселей

Было: 36,000 гауссианов × 8 px² = 288,000 px покрыто (13.9%)
Стало: 225,000 гауссианов × 12 px² = 2,700,000 px покрыто (130%)

Overdraw: 30% (гауссианы перекрываются)
```

**НО!** Гауссианы **не равномерны**:
```
Распределение по экрану:
┌─────────────────────────┐
│ ○ ○ ○ ○ ○ ○ ○           │  ← контуры: плотно
│ ○             ○           │
│ ○             ○           │  ← внутри: ПУСТО
│ ○             ○           │
│ ○ ○ ○ ○ ○ ○ ○           │  ← контуры: плотно
└─────────────────────────┘

Плотность на контурах: ~0.5 гауссиана/пиксель ✅
Плотность внутри: ~0.001 гауссиана/пиксель ❌
```

**Вывод**: Плотность увеличена, но **распределение неправильное** — всё на контурах.

## 🔬 Паттерн-анализ

### Что работает правильно:

1. ✅ **Проекция 3D → 2D**:
   - Перспектива корректна
   - Видна глубина сцены
   - Z-ordering работает

2. ✅ **Геометрическое распознавание**:
   - Форма арок читается
   - Пропорции сохранены
   - Структура Sponza узнаваема

3. ✅ **Цвет и материалы**:
   - Видны разные цвета (бежевый, коричневый, серый)
   - Материалы различимы

### Что НЕ работает:

1. ❌ **Заполнение поверхностей**:
   - Треугольники не заполнены
   - Видны только контуры
   - Просветы между точками

2. ❌ **Распределение гауссианов**:
   - Концентрация на рёбрах
   - Пустота внутри треугольников
   - Неравномерная плотность

3. ❌ **Визуальная непрерывность**:
   - Дискретные точки вместо поверхности
   - Нет smooth shading
   - Wireframe-эффект

## 🎯 Выводы из визуального анализа

### 1. Алгоритм работает **частично корректно**

**Что подтверждается**:
- ✅ Проекция ViewProjection работает
- ✅ Барицентрическое семплирование работает
- ✅ Gaussian рендеринг работает

**Что НЕ работает**:
- ❌ Семплирование даёт контуры, не поверхности
- ❌ Плотность недостаточна для заполнения
- ❌ Распределение неравномерное

---

### 2. Проблема фундаментальная, не параметрическая

**Увеличение параметров НЕ помогло**:
```
36K → 225K гауссианов: +520%
Размер: 15 → 80 screen multiplier: +433%

Результат: Всё равно контуры ❌
```

**Вывод**: Проблема в **методе**, а не в настройках.

---

### 3. Gaussian Splatting ≠ Surface Rendering

**Видимый паттерн**:
```
Gaussian Splatting показывает "ТОЧКИ"
Rasterization показывает "ПОВЕРХНОСТИ"
```

Это как разница между:
- **Пуантилизм** (рисование точками) 🎨
- **Традиционная живопись** (мазки кистью) 🖌️

Для Sponza нужна кисть, а не точки!
