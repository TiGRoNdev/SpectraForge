### 2. **Service Registration**

#### **Core Services (Singleton):**

```cpp
container.registerSingleton<ILogger, Logger>();
```

- **Logger**: Единственный экземпляр на всё приложение
- **Lifetime**: Singleton
- **Rationale**: Централизованное логирование для всего приложения

#### **Rendering Services:**

```cpp
// Renderer - Singleton
container.registerSingleton<IRenderer>([](Container&) {
    return std::make_shared<HybridFreGSRenderer>();
});

// Camera3D - Transient (новый экземпляр для каждого Engine)
container.registerTransient<Camera3D>([](Container&) {
    auto camera = std::make_shared<Camera3D>();
    camera->setPerspective(60.0f, 16.0f / 9.0f, 0.1f, 1000.0f);
    return camera;
});
```

- **Renderer**: Singleton (единственный рендерер)
- **Camera**: Transient (каждый Engine может иметь свою камеру)

#### **Application Components (P0.3) - Scoped:**

```cpp
// GameLoopManager - Scoped per Engine instance
container.registerScoped<GameLoopManager>([](Container&) {
    return std::make_shared<GameLoopManager>();
});

// WindowManager - Scoped per Engine instance
container.registerScoped<WindowManager>([](Container&) {
    return std::make_shared<WindowManager>();
});

// InputManager - Scoped per Engine instance
container.registerScoped<InputManager>([](Container&) {
    return std::make_shared<InputManager>();
});

// SceneCoordinator - Scoped per Engine instance
container.registerScoped<SceneCoordinator>([](Container&) {
    return std::make_shared<SceneCoordinator>();
});
```

- **Lifetime**: Scoped (один экземпляр на scope/Engine instance)
- **Rationale**: Каждый Engine должен иметь свои компоненты, но в рамках одного Engine они должны быть едиными

## 📈 Benefits of DI Integration

### **Before P0.7:**
```cpp
// Manual dependency management
Engine::Engine(const AppConfig& config, 
               std::shared_ptr<ILogger> logger,
               std::shared_ptr<IRenderer> renderer,
               std::shared_ptr<IResourceManager> resource_manager)
    : config_(config),
      logger_(std::move(logger)),
      renderer_(std::move(renderer)),
      resource_manager_(std::move(resource_manager)) {
    // Manual construction of components
    gameLoop_ = std::make_unique<GameLoopManager>();
    windowManager_ = std::make_unique<WindowManager>();
    inputManager_ = std::make_unique<InputManager>();
    sceneCoordinator_ = std::make_unique<SceneCoordinator>();
}
```

**Problems:**
- ❌ Tight coupling between Engine and concrete implementations
- ❌ Manual dependency passing through constructors
- ❌ Difficult to test in isolation
- ❌ Hard to swap implementations

### **After P0.7:**
```cpp
// DI-based dependency management
Engine::Engine(const AppConfig& config) : config_(config) {
    // Configure DI
    DISetup::configureDefault();
    
    // Resolve dependencies from container
    logger_ = DI::ServiceLocator::get<ILogger>();
    renderer_ = DI::ServiceLocator::get<IRenderer>();
    
    // Create scope for Engine-specific components
    container_->beginScope("engine_" + std::to_string(engineId));
    gameLoop_ = container_->resolve<GameLoopManager>();
    windowManager_ = container_->resolve<WindowManager>();
    inputManager_ = container_->resolve<InputManager>();
    sceneCoordinator_ = container_->resolve<SceneCoordinator>();
}
```

**Benefits:**
- ✅ Loose coupling через interfaces
- ✅ Централизованная конфигурация зависимостей
- ✅ Легко тестируется с mock implementations
- ✅ Простая замена implementations без изменения Engine

## 🚀 Usage Examples

### **Basic Usage (Default Configuration):**

```cpp
#include "SpectraForge/App/DISetup.h"
#include "SpectraForge/Core/DependencyInjection/Container.h"

int main() {
    // Configure DI with defaults
    SpectraForge::App::DISetup::configureDefault();
    
    // Get logger from global ServiceLocator
    auto logger = SpectraForge::Core::DI::ServiceLocator::get<ILogger>();
    logger->logInfo("Application started");
    
    // Get renderer
    auto renderer = SpectraForge::Core::DI::ServiceLocator::get<IRenderer>();
    
    return 0;
}
```

### **Advanced Usage (Custom Container):**

```cpp
#include "SpectraForge/App/DISetup.h"

int main() {
    // Create custom container
    auto container = std::make_unique<SpectraForge::Core::DI::Container>();
    
    // Configure with custom container
    SpectraForge::App::DISetup::configure(*container);
    
    // Create scope for Engine
    container->beginScope("engine1");
    
    // Resolve components
    auto gameLoop = container->resolve<GameLoopManager>();
    auto windowManager = container->resolve<WindowManager>();
    
    // Use components...
    
    // Cleanup scope
    container->endScope();
    
    return 0;
}
```

### **What Went Well:**

### **Challenges:**

### **Best Practices Established:**
📌 **Use ServiceLocator for Singletons**: Global access for shared services  
📌 **Use Scoped for Engine Components**: Proper cleanup per Engine instance  
📌 **Use Transient for Stateful Objects**: Camera, temporary objects  
📌 **Centralized Configuration**: One place (DISetup) for all registrations
