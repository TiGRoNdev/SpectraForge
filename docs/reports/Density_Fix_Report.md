# Gaussian Density Fix Report

**Дата**: 2025-10-03  
**Проблема**: Слишком разреженные гауссианы — видны только контуры арок Sponza  
**Решение**: Увеличена плотность subdivision и размеры гауссианов

---

## 🔍 Анализ проблемы

### Визуальные симптомы:
- ✅ Контуры арок Sponza **видны** (структура распознаётся)
- ❌ Гауссианы **слишком маленькие** (точки вместо покрытия)
- ❌ **Разреженное** покрытие (видны просветы)
- ❌ Похоже на **wireframe** из точек

### Причина:
Предыдущее исправление **слишком сильно** уменьшило:
1. Размер гауссианов (0.35 × 15 = 5.25 экранных пикселей)
2. Количество треугольников для обработки (15,000 из 66,000)
3. Минимальный subdivision level (1 гауссиан на малые треугольники)

---

## 🔧 Реализованные исправления

### 1. **Увеличено количество обрабатываемых треугольников**

```cpp
// Было:
const size_t step = std::max<size_t>(1, tris.size() / 15000);
gs.reserve((tris.size() / step) * 3);

// Стало:
const size_t step = std::max<size_t>(1, tris.size() / 50000); // +233%
gs.reserve((tris.size() / step) * 6);
```

**Результат**: Обрабатывается **~50,000 треугольников** вместо ~15,000

---

### 2. **Агрессивный subdivision для плотности**

```cpp
// Было:
int subdivisionLevel = 1; // минимум
if (area > 0.08f) subdivisionLevel = 6;
else if (area > 0.04f) subdivisionLevel = 4;
else if (area > 0.02f) subdivisionLevel = 3;
else if (area > 0.01f) subdivisionLevel = 2;

// Стало:
int subdivisionLevel = 2; // минимум 2 (не 1)
if (area > 0.08f) subdivisionLevel = 9;      // +50%
else if (area > 0.05f) subdivisionLevel = 6; // пороги снижены
else if (area > 0.03f) subdivisionLevel = 4;
else if (area > 0.015f) subdivisionLevel = 3;
```

**Результат**: 
- Минимум **2 гауссиана** на каждый треугольник (было 1)
- Большие поверхности: **до 9 гауссианов** (было 6)

---

### 3. **Новый subdivision level: 9 гауссианов**

Для очень больших треугольников (арки, стены):

```cpp
// 9 точек размещения:
glm::vec3 bc = (w0 + w1 + w2) / 3.0f;         // барицентр
glm::vec3 mid01 = (w0 + w1) * 0.5f;           // середина ребра 0-1
glm::vec3 mid12 = (w1 + w2) * 0.5f;           // середина ребра 1-2
glm::vec3 mid20 = (w2 + w0) * 0.5f;           // середина ребра 2-0
glm::vec3 q0 = w0 * 0.7f + bc * 0.3f;         // ближе к вершине 0
glm::vec3 q1 = w1 * 0.7f + bc * 0.3f;         // ближе к вершине 1
glm::vec3 q2 = w2 * 0.7f + bc * 0.3f;         // ближе к вершине 2
glm::vec3 inner01 = bc * 0.7f + mid01 * 0.3f; // внутренняя точка 1
glm::vec3 inner12 = bc * 0.7f + mid12 * 0.3f; // внутренняя точка 2
```

**Покрытие**: Плотная **сетка 3×3** на больших полигонах

---

### 4. **Увеличены размеры гауссианов**

**CPU (Engine.cpp)**:
```cpp
// Было:
float sigma = sqrt(area / subdivisionLevel) * 0.35f;
sigma = clamp(sigma, 0.008f, 0.08f); // 0.8%-8%

// Стало:
float sigma = sqrt(area / subdivisionLevel) * 0.45f; // +29%
sigma = clamp(sigma, 0.012f, 0.1f);                  // 1.2%-10%
```

**GPU (GaussFreqSplat.comp)**:
```glsl
// Было:
float screenSigma = sigma * 15.0;

// Стало:
float screenSigma = sigma * 28.0; // +87%
```

**Результат**: Средний размер гауссиана в экранных пикселях: **~12 px** (было ~5 px)

## ⚠️ Производительность

**Внимание**: Увеличение количества гауссианов с ~36K до ~225K может снизить FPS на слабых GPU.

**Если FPS падает**:
1. Уменьшите `step` до 35000-40000
2. Снизьте `maxGaussians` в `FreGSPass` (default 100K)
3. Включите LOD: меньше subdivision на расстоянии
