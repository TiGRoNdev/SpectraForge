# Foveation Effect Fix Report

**Дата**: 2025-10-03  
**Проблема**: Овал в центре экрана + слишком большие гауссианы  
**Решение**: Отключена фовеация + уменьшены размеры гауссианов

---

## 🔍 Обнаруженные проблемы

### 1. **Овал в центре экрана (Foveation Effect)**

**Причина**: В шейдере `GaussFreqSplat.comp` была активна **фовеация** — техника оптимизации для VR/AR:

```glsl
// До исправления:
if (!isInFovea(uv) && i > (gaussCount / 4)) {
    break;  // 75% редукция в периферии
}
```

**Эффект**: 
- В **центре экрана** (fovea) обрабатывались **все гауссианы**
- На **периферии** обрабатывалось только **25%** гауссианов
- Результат: **овальная область** высокой детализации в центре

**Решение**: ✅ Отключена фовеация — теперь **все гауссианы** обрабатываются **по всему экрану**

---

### 2. **Слишком большие гауссианы (Сферы вместо деталей)**

**Причина**: Неоптимальные параметры масштабирования:

```cpp
// До исправления (Engine.cpp):
float sigma = std::sqrt(area / subdivisionLevel) * 0.6f;
sigma = glm::clamp(sigma, 0.015f, 0.15f); // 1.5%-15%

// До исправления (GaussFreqSplat.comp):
float screenSigma = sigma * 50.0; // слишком большой масштаб
```

**Эффект**:
- Гауссианы покрывали **слишком большую область** экрана
- Результат: **видимые сферы** вместо плотного покрытия

**Решение**: ✅ Уменьшены параметры:
```cpp
// После (Engine.cpp):
float sigma = std::sqrt(area / subdivisionLevel) * 0.35f; // -42%
sigma = glm::clamp(sigma, 0.008f, 0.08f); // 0.8%-8% (было 1.5%-15%)

// После (GaussFreqSplat.comp):
float screenSigma = sigma * 15.0; // -70% масштаб
```

|----------|---------------|-------------------|-----------|
| **CPU sigma multiplier** | 0.6 | 0.35 | **-42%** |

## 💡 Что такое Foveation?

**Foveation** — это техника оптимизации для VR/AR, основанная на физиологии человеческого глаза:

### Принцип:
- **Fovea** (центральная ямка сетчатки) — область **острого зрения** (2° поля зрения)
- **Периферия** — область **низкого разрешения** (но высокой чувствительности к движению)

### Применение в рендеринге:
✅ **VR/AR**: Рендерить высокое качество только в центре взгляда (eye tracking)  
✅ **Экономия GPU**: До **60-70% производительности**  
❌ **Desktop 3D**: Не нужна (весь экран в фокусе)  

### Почему отключили:
- ❌ Нет eye tracking на desktop
- ❌ Пользователь смотрит на весь экран, не только центр
- ❌ Создаёт артефакт "овала"

---

## 📈 Влияние на производительность

### Фовеация (до):
- **Центр (25% экрана)**: 100% гауссианов → **высокая нагрузка**
- **Периферия (75% экрана)**: 25% гауссианов → **низкая нагрузка**
- **Средняя нагрузка**: ~43% от максимальной

### Без фовеации (после):
- **Весь экран**: 100% гауссианов → **равномерная нагрузка**
- **Средняя нагрузка**: 100% от максимальной

**Компенсация**: Уменьшение размера гауссианов **снижает overdraw** → производительность остаётся приемлемой
